Background

Brutus is a personal research project and might never make it to a final product. The motivation is to create a location transparent VM for shared-nothing actors. There are several opportunities to develop new garbage collection algoriths given this premisse.

Besides the implementation of a virtual machine there are also several opportunities to try new language features and different flavors in a type system.

Language

The Brutus language which does not even have a name yet is under heavy development. There are however already some key concepts.

  1. The language enforces a consistent coding style
  2. Written code must be easy to comprehend
  3. There is no null pointer
  4. There are no operators, therefore there is no precedence
  5. No packages but a module system
  6. \t and \r are illegal characters

Here is an example of how code might look like:

// Operators are left-associative by default and have no precedence.
// This is the same as writing val num = (1.+(3)).*(3) so the actual
// value is 12.
val num = 1 + 3 * 3 

// A function declaration. Anonymous functions are only
// allowed as arguments for higher order functions.
//
// The if expression is also different in the sense
// that it combines if and else-if constructs you
// already know from other languages.
// _ is the wildcard and means else.
//
// "yes" is a synonym for "true" and "no" is a synonym for "false".
//
// Parameter and return types are always required in order to
// make sure type-inference does not do any harm.
fun isEven(x: Int): Boolean = 
  if {
    x % 2 == 0 -> yes
    _ -> no
  }

// This will print "12 is even"
if isEven(num) -> Console.println("$num is even.")

// Operators ending with : are right-associative so this
// is the same as writing ((Nil.::(3)).::(2)).::(1)
val list = 1 :: 2 :: 3 :: Nil

// Multiply the contents of the list by two.
// Console#println is a reference to the println method
// of the Console object.
list map { x -> x * 2 } map { _ * 2 } foreach Console#println

// Classes are final by default so we have to
// declare them virtual to allow inheritance.
virtual class A(val prop: Int) {
  // Methods are also final by default so we
  // have to declare them virtual if another
  // class is allowed to overwrite them.
  //
  // The foo method is not required to specify
  // the return type because the missing equality
  // sign declares it already as void.
  virtual fun foo() {
    println("Hello World")
  }
}

// B inherits from A, we pass the value 123
// as the constructor argument
class B : A(123) {
  override fun foo() {
    println(prop)
  }
}

// Create an instance of B and call the foo method
val b = new B()
b.foo() //prints 123

// The notation x#y is always a reference to the member.
// In case of b#field the type of lens is Property[Int] and
// allows us to abstract over arbitrary properties.
val lens = b#prop

fun doubleIt(x: Property[Int]) {
  val oldValue = x.value
  val newValue = oldValue * 2
  x.update(newValue)
}

doubleIt(lens)

b.foo() //246

Development

Building Brutus from Source

The development environment for Brutus is generated using GYP. Currently supported platforms include Linux and Windows (64bit).

Because this project is also aimed at a better understanding of the new C++11 features a modern compiler is required. The project compiles fine using clang on Linux and Visual Studio 2012 Express (Desktop Edition) on Windows.

Prequisites

Besides a working compiler tool chain you will need a Python and Subversion installation to download and run GYP.

  • Python 2.7.3
  • Subversion

Linux

  1. make dependencies to download GYP
  2. make x64.debug or make x64.release to compile an executable.
  3. ./recompile-on-change.sh can be used if inotify-tools are available

Windows

  1. Install GYP via svn checkout --force http://gyp.googlecode.com/svn/trunk build/gyp --revision 1451
  2. python build/gyp_brutus to generate the project files
  3. Open in Visual Studio 2012
